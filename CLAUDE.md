# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run build    # generate index.html from src/index.html + projects.json
npm run dev      # build then serve locally (npx serve .)
node build.js --refresh-fonts  # force re-download Inter font (bypasses cache)
```

## Architecture

This is a zero-dependency static site with a custom build step. There are two source files and one build script:

- **`src/index.html`** — template; contains all CSS, layout, and JS inline. The `render()` function takes a projects array and stamps out app icons. During dev (no build), the script falls back to `fetch('projects.json')`. The comment pair `/* __DATA_LOAD_START__ */ ... /* __DATA_LOAD_END__ */` is the injection point.
- **`projects.json`** — data source; each entry has `name`, `label` (single char shown on icon), `description`, `color` (hex), `demo` URL, `github` URL, and `wip` (bool).
- **`build.js`** — Node.js script (no npm deps) that: (1) reads `projects.json` and inlines it into the template replacing the fetch call, (2) downloads Inter from Google Fonts and inlines all latin `@font-face` blocks as base64 data URIs, removing the Google Fonts `<link>` tags. Font data is cached at `src/.fonts.cache.css` so subsequent builds are instant.

The built output is **`index.html`** at the repo root (gitignored; generated by CI).

## Deployment

GitHub Actions (`.github/workflows/deploy.yml`) runs on every push to `main`: builds `index.html`, copies it to `dist/`, and deploys to GitHub Pages via the artifact upload/deploy actions. GitHub Pages source must be set to **GitHub Actions** in repo settings.

## Adding a project

Edit `projects.json` — no code changes needed. The build will pick it up automatically.
